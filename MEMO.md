# コンテニュー連打の必要回数計算

## 前提

- ゲームは30fps固定
- タイトル画面で「コンテニュー → セーブデータ選択 → Bで戻る」の1サイクルで約**700フレーム**進む
  - これは画面遷移アニメーションによるフレーム消費であり、待機時間は含まない
- ボタンを押していない待機時間中も30fps でフレームが進み続ける

## 1サイクルあたりのフレーム数

```
1サイクルのフレーム数 = 700（遷移分）+ 待機秒数 × 30（時間経過分）
```

### 例

| 連打速度 | 待機時間 | 1サイクルのフレーム数 |
|---|---|---|
| 人間 80BPM（1.33回/秒） | 0.75秒 | 700 + 22.5 = **722F** |
| 1回/秒 | 1秒 | 700 + 30 = **730F** |
| 機械 2回/秒 | 0.5秒 | 700 + 15 = **715F** |
| 機械 3回/秒 | 0.33秒 | 700 + 10 = **710F** |

## 必要な押し回数と時間

```
必要な押し回数 = 目標フレーム ÷ 1サイクルのフレーム数
必要な時間（秒）= 必要な押し回数 ÷ 連打速度（回/秒）
```

### 目標フレーム 9,673,507 の場合

| 連打速度 | 押し回数 | 所要時間 |
|---|---|---|
| 人間 80BPM | 13,388回 | 約2.8時間 |
| 1回/秒 | 13,250回 | 約3.7時間 |
| 機械 2回/秒 | 13,529回 | 約1.9時間 |
| 機械 3回/秒 | 13,624回 | 約1.3時間 |

## 備考

- 1サイクル700フレームは推定値。実機で動画撮影して計測した値に差し替えること
- 700フレームの計測方法：タイトル画面からコンテニューを押してBで戻るまでの動画をフレーム単位で数える
- マイコン（Arduino Leonardo / Pro Micro）でボタン入力を自動化する場合、SwitchのUSB HIDとして認識させる
